3D Affordance Grounding 系统
基于记忆增强的机器人可供性定位与交互推理系统

系统概述

本系统围绕 3D Affordance Grounding (3D 可供性定位) 任务构建，集成了四大核心子系统，实现从2D图像交互到3D点云推理的端到端智能处理。

🎯 主推理引擎

IAG/IAG_TextEmb模型，输入图像+点云+边界框，输出逐点可供性概率和17类分类logits

🧠 偏好记忆系统

在交互中累积per-point偏好矩阵，通过相似检索+跨实例对齐增强后续推理

🖼️ 图像记忆系统

以(object, affordance)为复合键缓存图像ROI特征，在JRA之前完成特征平均增强

📝 智能标注模型

自动生成subject_box/object_box/affordance label，替代纯人工标注

技术架构

推理模型原理

复现论文 "Grounding 3D Object Affordance from 2D Interactions in Images"，实现6阶段数据流：

输入数据

  • img [B,3,H,W] - 含有交互主体与客体的RGB图像
  • xyz [B,3,2048] - 待预测的3D点云
  • sub_box/obj_box [B,4] - 主体/客体边界框

处理流程

  1. Img_Encoder - ResNet-18提取图像特征，ROI对齐获取F_i/F_s/F_e
  2. Point_Encoder - PointNet++ MSG下采样2048→64点，提取F_p
  3. JRA - 联合区域对齐，双向cross-modulation融合图像-点云特征
  4. ARM - 可供性揭示模块，cross-attention生成affordance特征
  5. Decoder - PointNet++ FP上采样，输出逐点概率和分类logits
  6. TextEmb - 可选文本嵌入，支持动作语义切换

记忆系统架构

设计哲学：让模型"在不重新训练参数的前提下，从过去成功/失败的交互中受益"。

📊 MemoryIndexer

ARM特征→128-D索引向量，监督对比损失训练，拉近同类可供性

🔍 MemoryRetriever

FAISS近邻搜索+多谓词过滤(affordance/outcome/reward)

🔄 MemoryAligner

多头注意力跨实例对齐，历史偏好迁移到当前点云

⚡ MemoryFusion

softmax加权融合+时间衰减，残差加到原始logits

双路存储

  • 向量路径 - FAISS IndexFlatIP(128)内积检索
  • 结构路径 - SQLite表存储完整MemoryEntry
  • 淘汰策略 - LRU按(access_count, timestamp)升序淘汰
